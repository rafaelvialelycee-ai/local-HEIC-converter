Behind the scenes

How does local conversion work?

Unlike most online converters that upload your files to a remote server, this tool uses WebAssembly (a near-native binary format that runs inside modern browsers) to process images entirely on your machine. Your photos never travel over the internet — they're read, decoded and re-encoded directly in your browser's memory.

01

HEIC & HEIF decoding

HEIC (High Efficiency Image Container) is Apple's default photo format since iOS 11. It offers superior compression compared to JPEG — roughly 2× smaller at the same quality — but isn't natively supported by Windows or older web apps. This converter uses the open-source heic2any library, which decodes the HEVC-encoded content in your browser via WebAssembly.

02

WebP conversion

WebP is a modern image format developed by Google, offering both lossy and lossless compression. While widely supported in browsers, WebP images can't always be opened in native OS apps or sent to clients expecting JPEG. This tool decodes WebP images using the canvas API and re-encodes them as JPEG at 95% quality — preserving maximum detail.

03

Privacy by design

Every file you select stays on your device. The conversion happens inside a sandboxed browser tab, with zero network requests for your image data. This makes the tool ideal for sensitive files — personal photos, medical images, confidential documents — where cloud processing is unacceptable.

04

Batch download as ZIP

Convert dozens of files in a single batch and download them all as a ZIP archive — no manual downloading of individual files. The ZIP is assembled client-side using JSZip and offered as a direct browser download, again with no server involvement.

Format guide

What is the HEIC format?

HEIC stands for High Efficiency Image Container. It was introduced by Apple in 2017 with iOS 11 and macOS High Sierra as the default capture format for iPhone and iPad cameras. The format uses the HEVC (H.265) video codec to store still images, achieving roughly 50% smaller file sizes compared to JPEG — without any visible loss in quality.

The problem? HEIC is not universally supported. While Apple devices open it natively, Windows 10/11 requires a paid codec extension, most image editors don't support it out of the box, and many web platforms and email clients only accept JPEG or PNG. This is why converting HEIC to JPG is one of the most common image tasks for iPhone users.

When should you keep HEIC? If you're staying within the Apple ecosystem — editing in Photos, sharing via AirDrop, or archiving on iCloud — HEIC is excellent. Convert to JPG only when you need cross-platform compatibility.

FAQ

Frequently asked questions

Are my files uploaded to a server?

No. All processing happens entirely in your browser using JavaScript and WebAssembly. Your files are read from your local disk into browser memory, converted, and offered as a download — no data is ever sent over the network. You can even disconnect from the internet after the page loads and the converter will still work.

Why does HEIC conversion fail on Safari?

Safari on iOS has strict limitations on JavaScript memory and WebAssembly execution. HEIC files encoded with Live Photo metadata or HDR/Dolby Vision can exceed these limits and cause the library to crash silently. For best results, use Chrome or Edge on desktop. If you must use Safari, try converting smaller batches (5 files at a time).

What quality is the output JPG?

All output JPGs are encoded at 95% quality, which is visually lossless for most practical purposes. This is a deliberate balance between file size and fidelity. The conversion process is: decode original → draw on an HTML5 Canvas → re-encode as JPEG at 0.95 quality.

Is there a file size or quantity limit?

There is no artificial limit imposed by the tool. The practical limit is your browser's available RAM. Typically, you can convert 30–50 high-resolution iPhone photos (3–5 MB each) in a single batch without issues on a modern machine. Very large files (50 MB+) may be slow or fail on low-memory devices.

Does this work on iPhone or Android?

Yes, the converter works on mobile browsers. However, mobile devices have significantly less JavaScript memory available, so we recommend converting no more than 5–10 files at a time on mobile. On Android, Chrome is the most reliable choice.

Can I convert WebP to JPG as well?

Yes. The converter accepts .webp files in addition to .heic and .heif. WebP conversion uses the browser's native canvas API, which is extremely fast and reliable across all modern browsers. PNG and standard JPEG files are also accepted and will be re-encoded at 95% JPG quality.
